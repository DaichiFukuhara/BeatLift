## サマリー

現代のトレーニングアプリは、単なる「デジタル記録帳」から、データ分析・AI プログラム生成・ウェアラブル連携を備えた **デジタルコーチ** へと進化している。一方でユーザーの要求は二極化しており、

- **記録スピード重視層**（ジムでの摩擦最小化）
- **科学的プログラム自動化層**（漸進性過負荷の自動計算）
- **コミュニティ重視層**（SNS によるモチベーション維持）

の三つに大別される。本資料ではこの三軸を意識しつつ、12 本のアプリを Pros / Cons とターゲット層で整理する。

---

## 1. iPhone で人気のアプリ

iOS 市場では、**直感的な UI** と **Apple Watch とのシームレス連携** が評価を決定づける。

### Strong

フリーウェイト記録のデファクトスタンダードとして長年君臨してきたアプリ。

**主な特徴**

- 種目・重量・回数・セット数を最小タップで入力できる UI
- Apple Watch 連携でインターバル管理と記録が手首で完結
- 「純粋なロギング」に最適化された機能美

**Pros**

- 洗練されたクリーンな UI、認知負荷の低さ
- 起動からセット記録までが極めて高速
- Reddit 等では「最もクリーンなデザイン」「3 年使った進捗管理のゲームチェンジャー」と評される

**Cons**

- **開発の停滞**：数年単位で意味のある新機能が追加されておらず、「サブスクだけ徴収している」との批判が多い
- **Apple Watch 版の改悪**：デジタルクラウンでの数値入力が削除され、Watch から過去履歴が見られなくなった
- **オートフィルのバグ**：前回値ではなくランダムな過去値を引いてくる事例あり
- 無料版はルーティン保存数が **3 つまで** で分割法ユーザーには厳しい

**ターゲット層**：既存メニューを淡々とこなしたい層

---

### Hevy

Strong の代替として急速にシェアを伸ばしている新興アプリ。

**主な特徴**

- 純粋な記録機能 + ユーザー間の SNS 機能
- トレーニングボリュームの推移、部位別バランスのヒートマップ
- Web 版併設で PC からプログラム構築可能

**Pros**

- 「Strong v3」と称されるほど現代的な UI
- 活発なアップデート、コミュニティフィードバックの取り込みが速い
- 無料版でも**ワークアウト記録は無制限**
- 有料版に**買い切り（ライフタイム）プラン**があり、サブスク疲労層に強く響く

**Cons**

- SNS 要素を「ノイズ」と感じる純粋ロガー層もいる
- 無料版のカスタムルーティン保存は **4 つまで**（PPL スプリット等を週 5–6 回回す層は有料版必須）
- Apple Watch 同期は概ね良好だが、約 10% の確率で同期エラーの報告あり

**ターゲット層**：記録を共有したい現代的トレーニー

---

### JEFIT

1,400 種目以上の巨大データベースを持つ古参アプリ。

**主な特徴**

- 圧倒的な種目データベース
- 1RM 推移・部位別疲労の詳細分析
- AI ベースのプログレッシブ・オーバーロード提案機能

**Pros**

- 中・上級者のピリオダイゼーション管理に強い
- 長年蓄積されたコミュニティとプログラム資産

**Cons**

- **2014 年頃から大きく変わっていない時代遅れの UI**：ジムでの直感的なメニュー変更が困難
- **AI 適応機能の暴走**：長年保存していたユーザーのワークアウトプランが勝手に書き換えられ、元に戻せない仕様変更で有料会員からも「事実上使用不可」との声
- Apple Watch アプリのクラッシュ・同期エラーが長期間放置

**ターゲット層**：緻密なデータ分析を好む上級者

---

### iPhone 系まとめ

| アプリ     | 主な特徴                    | Pros                       | Cons                                   | ターゲット         |
| ---------- | --------------------------- | -------------------------- | -------------------------------------- | ------------------ |
| **Strong** | 直感的 UI、Apple Watch 連携 | 洗練デザイン、記録の速さ   | 開発停滞、Watch の機能後退             | 淡々と記録したい層 |
| **Hevy**   | SNS、Web 版、ヒートマップ   | 活発な開発、買い切りプラン | 無料枠ルーティン制限、SNS のノイズ感   | 現代的トレーニー   |
| **JEFIT**  | 1,400+ 種目 DB、AI 提案     | 膨大な分析データ           | UI が古い、AI による既存プラン破壊バグ | 上級分析志向       |

## 2. Android で人気のアプリ

Android 市場では、**カスタマイズ性**・**軽量さ**・**多様なウェアラブル連携**（Wear OS / Garmin）が重視される。

### FitNotes

Android 専用のミニマリスト記録アプリ。

**主な特徴**

- 広告ゼロ、完全オフライン動作
- アカウント登録不要、ローカル or 個人 Google Drive 保存
- フリーウェイトの重量・回数記録に完全特化

**Pros**

- **動作の軽さが圧倒的**：通信環境の悪い地下ジムでも遅延ゼロ
- プライバシー保護が徹底
- コミュニティから「ワークアウトアプリの王様」と評され、開発者支援のために有料版（3–5 ドル）をあえて買うユーザーが多い

**Cons**

- 種目の動画・イラスト解説が皆無 → **経験者前提**
- デフォルトでセット完了マークが OFF など、初期設定が直感的でない
- プログレッシブ・オーバーロード自動計算等の現代的提案機能はゼロ

**ターゲット層**：通信・電池を節約したいベテラン層

---

### Progression

Android のマテリアルデザインに最適化されたアプリ。

**主な特徴**

- 過去履歴を参照しながらの重量設定
- **種目ごとに個別のインターバル時間**を設定可能（コンパウンドは 3–5 分、アイソレーションは 1.5 分など）
- 個人開発者によるサポートが密

**Pros**

- 過去履歴の参照シームレス
- 開発者（Zoltan 氏）からのメール返答が数時間以内、品質改善サイクルが速い

**Cons**

- 個人開発ゆえのリソース不足：大規模アップデートで **データベース同期エラー → 履歴消失** の致命的バグ報告あり
- 種目選択画面のラグ報告（後に修正）
- AI による漸進性過負荷の組み込みは不足

**ターゲット層**：美しい UI で柔軟に自己管理したい層

---

### GymRun

Wear OS / Garmin 連携が Android 市場で最強クラスのアプリ。

**主な特徴**

- スマートフォンを置いたまま、**ウォッチ単独でルーティン・スーパーセット・インターバル管理**を完結
- Garmin の純正筋トレ計測（カーディオ偏重）の弱点を補完するプラグインを提供

**Pros**

- ネイティブなウォッチ連携の完成度
- Garmin ユーザーには正確なセット管理ツールとして重宝される

**Cons**

- ウォッチ連携機能は **プレミアム版必須**
- Garmin 側の仕様で、ウォッチ本体に保存できるワークアウトは **25 個まで**。それを超える場合はカレンダー同期等のワークアラウンドが必要
- トレーニング中の予定外種目追加・順序変更の UI が直感的でない

**ターゲット層**：手首のデバイスだけで全記録を完結させたい層

---

### Android 系まとめ

| アプリ          | 主な特徴                           | Pros                       | Cons                             | ターゲット       |
| --------------- | ---------------------------------- | -------------------------- | -------------------------------- | ---------------- |
| **FitNotes**    | 広告なし・オフライン・最軽量       | 動作の軽さ、プライバシー   | 動画ガイドなし、自動化ゼロ       | 節約志向ベテラン |
| **Progression** | マテリアルデザイン、種目別タイマー | 履歴参照容易、密なサポート | 履歴消失バグ、自動化不足         | 柔軟な自己管理層 |
| **GymRun**      | Wear OS / Garmin ネイティブ連携    | スマホ不要の完結体験       | 連携の有料化、ウォッチ側保存制限 | 手首完結志向     |

## 3. 日本国内で人気のアプリ

国内ユーザーは、**完全日本語対応**・**kg ベース**・**継続を促す視覚的フィードバック**を重視する。

### 筋トレ Memo

国内で広く普及しているシンプルな記録アプリ。

**主な特徴**

- 種目・セット数・回数・重量を数タップで入力
- カレンダー上で **トレーニング部位を色分け表示**（胸＝赤、背中＝青など）

**Pros**

- 初心者の参入障壁が極めて低い
- 部位サイクル管理を視覚的に直感把握できる

**Cons**

- **VALX コラボに伴う大型アップデート以降の UI 改悪**に既存ユーザーから激しい反発
- 週 / 月の総ボリューム比較グラフ、部位別ビジュアル表示などが消滅
- メニュー編集中・種目削除中のクラッシュで **その日の記録が丸ごと消失** する深刻なバグ報告

**ターゲット層**：視覚的サイクル管理を重視する国内初心者

---

### Library

1RM 自動計算と BIG3 推移グラフに強い、SNS 機能統合型アプリ。

**主な特徴**

- 1RM 自動計算、BIG3（ベンチ・スクワット・デッドリフト）合計のグラフ化
- アプリ内タイムラインで他ユーザーのトレーニングを閲覧、リアクション可
- ランニング記録もタグで管理可

**Pros**

- 片手でサクサク記録できる操作性
- **SNS によるモチベーション維持**：孤独になりがちな筋トレで国内トレーニー同士の相互刺激を生む

**Cons**

- 小規模開発ゆえのアップデート停滞期間が長い（開発者自身も「アップデートしないとと思いながら時間が過ぎた」とコメント）
- 細かい UI 変更で「使い勝手が変わって戸惑う」既存ユーザーの声

**ターゲット層**：他者からの刺激でモチベーションを保ちたい層

---

### ハルク系（HulkFitness 等）

トレーニングだけでなく **体組成も統合管理** するタイプの総合アプリ群。

**主な特徴**

- 体重・体脂肪率・筋肉量・水分摂取・食事マクロを一元管理
- カレンダーで長期推移を比較

**Pros**

- **挙上重量と身体の変化の相関**を一つのアプリで分析可能
- バルクアップ / 減量期の長期ロードマップを描きやすい

**Cons**

- 多機能化により情報量が多く、ジム中の即時入力には不向き
- アプリによってはパーソナルコーチング機能（ビデオ通話・特定ジム連携等）が前面に出すぎ、純粋ロガー層には「画面上のノイズ」になる

**ターゲット層**：増量・減量など長期的な身体変化を追跡したい層

---

### 国内系まとめ

| アプリ          | 主な特徴                       | Pros                               | Cons                                     | ターゲット       |
| --------------- | ------------------------------ | ---------------------------------- | ---------------------------------------- | ---------------- |
| **筋トレ Memo** | カレンダー部位色分け、簡単入力 | 導入の容易さ、視覚的疲労管理       | コラボ後の UI 改悪、データ消失バグ       | 国内初心者       |
| **Library**     | 1RM 自動計算、BIG3 グラフ、SNS | SNS によるモチベーション、片手操作 | 開発の停滞、UI 変更の波                  | 刺激を求める層   |
| **ハルク系**    | 体組成も統合管理               | 出力と身体変化の相関可視化         | 多機能ゆえの煩雑さ、過剰なコーチング機能 | 長期身体改造志向 |

## 4. 海外で人気のアプリ

海外市場では、**スポーツ科学に基づくアルゴリズム最適化**・**著名プログラムのテンプレ実行**・**AI フォーム解析**など、技術的に高度な機能が求められる。

### Boostcamp

nSuns、GZCLP、5/3/1、Reddit PPL など**海外の著名ストレングスプログラム**をアプリ内でそのまま実行できるアプリ。

**主な特徴**

- スプレッドシート管理からの解放
- 現在の 1RM を入力するだけで、数週〜数ヶ月先のセット・重量・**ディロード週**まで自動算出
- **プレート計算機能**：バーベルに何キロのプレートを何枚付ければよいかを指示

**Pros**

- 「アプリの指示通りに挙げるだけ」の **認知負荷ゼロ** 体験
- 科学的根拠ある人気プログラムへのシームレスアクセス

**Cons**

- **Pro 版が年間 80 ドル前後と高額**：「元々無料の Reddit プログラムに払う額か」との議論が絶えない
- レストタイマーが画面を占有 → 次種目が確認しづらい（バックグラウンド + 音声通知が望まれる）
- 1 日だけ種目を入れ替えるとプログラム全体に影響する硬直性

**ターゲット層**：自分でメニューを組まず、実績ある計画に従いたい層

---

### Caliber

科学的アプローチに基づく総合トレーニングアプリ。

**主な特徴**

- 記録から **ストレングススコア**（同年齢・同体重層との相対筋力）を算出
- 500+ の高品質エクササイズ動画
- パーソナライズされたプランニング

**Pros**

- 広告なしの無料枠が高品質
- **ストレングススコアによるゲーミフィケーション**で初〜中級者のモチベーションを劇的に上げる
- フォーム指導の理論的解説が詳細

**Cons**

- パーソナルトレーナー指導つきの「Premium」プランが **月額 199 ドル** と超高額
- 画面内ホワイトスペースが多くスクロール量が多い
- ダークモードの実装が遅れた（後に対応）
- ドロップセット・スーパーセットの記録に複数画面遷移が必要

**ターゲット層**：科学的指標で成長を測りたい初〜中級者

---

### Gravitus

海外のパワーリフター・シリアストレーニーから強い支持を集める競技志向アプリ。

**主な特徴**

- **AI コンピュータービジョン**でバーパス（バー軌道）・しゃがみの深さ・レップスピードを解析
- リーダーボードで他ユーザーと挙上重量を競う SNS 要素

**Pros**

- レップスピード（= 中枢神経系の疲労指標）の客観評価は**怪我予防と限界更新**に高い価値
- 競技的モチベーションを引き出すリーダーボード

**Cons**

- AI 解析活用には毎セットの **カメラセットアップ・録画開始/終了** が必要
- 結果としてワークアウト全体が **1.5–2 分間延び**するトレードオフ
- 種目別カスタムレストタイマーが無いなど、UI 細部が痒い所に届かない

**ターゲット層**：フォーム改善や重量更新に命を懸けるパワーリフター

| **Boostcamp** | Reddit 発の著名プログラムを実装 | プログレッシブ・オーバーロードの完全自動化 | サブスク高額、UI の硬直性               | 実績プログラム実行志向 |
| **Caliber**   | ストレングススコア、動画指導    | 科学的指標による客観評価、良質な無料枠     | コーチング 199 ドル/月、UI ステップ多い | 科学的成長指標志向     |
| **Gravitus**  | AI バー軌道・速度解析、競技要素 | フォームの客観評価                         | 撮影による時間増、種目別タイマー不足    | パワーリフター         |

---

## 5. 横断トレンド分析

12 本のユーザーフィードバックを総合すると、**4 つの構造的トレンド**が浮かび上がる。

### 5.1 「認知負荷の排除」vs「アルゴリズム主導のガイダンス」

UI 設計には明確な二極構造がある。

| ミニマリズム陣営           | アルゴリズム陣営                       |
| -------------------------- | -------------------------------------- |
| FitNotes / Strong          | JEFIT (Adaptive) / Boostcamp / Caliber |
| 「デジタルな紙とペン」     | 「次に挙げる重量を AI が指示」         |
| 入力時間・認知負荷を極小化 | 計画立案の負荷を AI に肩代わりさせる   |

**因果分析**

- JEFIT の AI アップデートに対する激しい反発は、**中・上級者がオートレギュレーション（その日の体調に応じた自己調整）を AI に奪われることを嫌う** ことを示している
- 一方、Boostcamp が初心者に絶賛されるのは、「次回何 kg を何回」の迷いを完全に消す価値が大きいから

**今後の方向**：**AI 委任モードとマニュアル入力モードをシームレスに切り替えられる**ハイブリッド設計が市場で生き残る条件になる。

---

### 5.2 サブスク疲労と「コミュニティ」という新たな付加価値

国境を問わず最も多い不満は **「基本的な記録機能への高額月額課金」**。本質的に Excel の代替に過ぎない機能に年間数十〜100 ドル超を払い続けることへの限界が来ている。

**勝者の戦略**

- **FitNotes**：完全無料 + 数ドルの寄付的買い切り
- **Hevy**：記録は無料 + SNS（ネットワーク効果） + **買い切りプラン**

**示唆**：純粋なロギング単体でのマネタイズはもう成立しない。今後の課金モデルが成立するのは、

1. **AI による動作解析**（Gravitus 型）
2. **ネットワーク効果**（Hevy / Library 型のコミュニティの熱量）

のいずれかを提供できるアプリだけになる。

---

### 5.3 ウェアラブル連携は「付加価値」から「必須インフラ」へ

汗ばんだ手でスマホを取り出してロック解除する行為自体が、UX の摩擦として認識され始めた。

- **Strong** が Apple Watch のデジタルクラウン入力を廃止 → Hevy / GymRun への移行を加速
- **JEFIT** の Watch アプリのクラッシュ放置 → 同様の流出
- **GymRun** が Android で評価される理由は、**スマホから完全独立したスタンドアロン体験**を Garmin / Wear OS で提供しているから

**示唆**：今後の主戦場は「スマホの大画面 UI」から、**「ウォッチの極小画面で摩擦なく入力を完結させ、触覚・音声で的確にフィードバックする」**ことへシフトする。

---

### 5.4 身体変化の総合トラッキングへの回帰

ハルク系アプリが示すように、**挙上重量 + 体重 + 体脂肪率の統合トラッキング**の重要性が再認識されつつある。

筋肥大・減量は最終的に栄養と休養に依存するため、

> 「ベンチ 100 kg 達成」が**バルクアップによる結果**なのか、**体重維持下の純粋な神経適応**なのか

を相関的に分析できるプラットフォームが求められている。

**今後の方向**：**HealthKit / Google Fit 経由でスマート体重計・睡眠データ・カロリーデータを自動統合**し、トレーニング出力と体組成変化の相関を AI が分析する機能が次世代スタンダードになる。

---

## 6. BeatLift への示唆

本資料の競合分析を、BeatLift の設計判断にマッピングすると以下の通り。

### 6.1 既に正しい方向

| BeatLift の設計                          | 競合分析からの裏付け                                                  |
| ---------------------------------------- | --------------------------------------------------------------------- |
| ローカル SQLite 完結・サーバーなし       | FitNotes が「動作の軽さ・プライバシー」で熱狂的に支持される構造と一致 |
| 楽観的更新による即時 UI 反映             | Strong / FitNotes の **認知負荷ゼロ** 思想と整合                      |
| 種目ごとに前回値を引き継ぐ追加挙動       | Progression の「過去履歴の参照容易さ」評価軸と一致                    |
| Epley 式による推定 1RM 表示              | Library の評価軸（1RM 自動計算）に対応                                |
| 1 日 1 セッション（`date` UNIQUE）       | 筋トレ Memo のカレンダー粒度と一致、迷いを生まない                    |
| HealthKit + AirPods Pro 3 心拍連携の予定 | 5.3 の「ウェアラブルが必須インフラ化」トレンドに正対                  |

### 6.2 検討すべき機能・優先度

| 検討事項                                       | 根拠                                                                      | 優先度               |
| ---------------------------------------------- | ------------------------------------------------------------------------- | -------------------- |
| **種目ごとの個別レストタイマー**               | Progression の高評価機能、現状 BeatLift は既定 90 秒のみ                  | 中                   |
| **部位別の視覚的サマリ**（カレンダー色分け）   | 筋トレ Memo の最大の支持理由、移行ユーザーへの訴求になる                  | 高（移行体験に直結） |
| **Apple Watch 対応**                           | 5.3 のトレンド、Strong からの離反理由そのもの                             | 高（ただし後期）     |
| **ボリューム推移グラフ・部位別バランス可視化** | Hevy のヒートマップ、筋トレ Memo の改悪で失われた機能                     | 中                   |
| **体組成統合（HealthKit 経由）**               | 5.4 の総合トラッキングトレンド                                            | 低〜中               |
| **データのバックアップ / エクスポート**        | 筋トレ Memo・Progression のデータ消失バグでユーザー信頼が崩れた事例       | 高                   |
| **SNS / コミュニティ機能**                     | Hevy / Library の差別化軸だが、個人開発 + オフライン哲学とは合わない      | 見送り推奨           |
| **AI プログラム自動生成**                      | Boostcamp の支持軸だが、中上級者の反発リスク（5.1）。マニュアル併用が前提 | 後期                 |

### 6.3 避けるべき罠

- **JEFIT 型の「AI が既存プランを勝手に書き換える」体験**：ユーザーの裁量を奪う設計は中上級者に強く嫌われる
- **多機能化による情報過多**（ハルク系の Cons）：ジム中の即時入力という主目的を曇らせない
- **筋トレ Memo 型のクラッシュ → データ消失**：マイグレーションとエラー時のフォールバックは早期に堅牢化すべき
- **基本機能への高額サブスク**（Strong / Caliber の Cons）：個人プロジェクトとして当面無料前提だが、将来課金するならコミュニティか AI 解析を伴うべき

## 7. 結論

トレーニング管理アプリ市場における最適解は、ユーザーセグメントごとに完全に分かれている。

| セグメント                               | 現在の最適解                           |
| ---------------------------------------- | -------------------------------------- |
| 純粋なロギング効率を求めるベテラン       | Strong（iPhone） / FitNotes（Android） |
| コミュニティとマルチプラットフォーム同期 | Hevy                                   |
| 視覚的わかりやすさ（国内ライト）         | 筋トレ Memo（ただし改悪・バグに注意）  |
| 世界標準の科学プログラムに従いたい       | Boostcamp / Caliber                    |
| フォーム改善とハードな競技性             | Gravitus                               |

最終的に、**最適なアプリとは「ユーザーのフィットネスリテラシー」と「ジムでの集中力を阻害しない UX の許容度」が完全に一致するもの**である。

開発者にとっての生存戦略は、

1. **データ自動化（AI 介入）と個人裁量（マニュアル操作）のバランス**を慎重に見極める
2. **ウェアラブル端末との統合**を深化させる

の 2 点に集約される。BeatLift はオフライン完結・記録の速さという FitNotes / Strong 軸の強みを土台に、AirPods Pro 3 心拍連携で **5.3 のウェアラブル必須インフラ化トレンドに正対する** ポジションを取れる。これは現時点で市場のホワイトスペースに近い。
